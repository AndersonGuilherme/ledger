---
description: Run the discovery phase for a feature, module, or initiative
argument-hint: <feature-or-scope>
---

Use the **master-orchestrator** to run the **Discovery phase** for: **$ARGUMENTS**.

Follow this workflow:

1. First identify whether the request is truly in discovery phase.
2. Coordinate the correct agents for discovery:
   - product-analyst
   - product-owner-orchestrator
   - financial-domain-expert if money, balances, billing, invoices, transfers, or projections are involved
   - analytics-engineer if tracking, KPIs, funnels, or metrics are central
3. Produce a structured discovery outcome with:
   - problem definition
   - objective
   - scope
   - out of scope
   - user flow
   - business rules
   - edge cases
   - acceptance criteria
   - open decisions
   - prioritized backlog seed
4. Detect and explicitly list any missing information.
5. Do **not** jump to implementation.

Output format:

### Current Phase
Discovery

### Agents Involved
List which agents were consulted and why.

### Discovery Output
Provide the structured discovery.

### Missing Pieces
List anything still unclear.

### Next Step
Recommend the correct next command.
