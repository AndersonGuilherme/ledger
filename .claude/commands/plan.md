---
description: Break an approved feature or architecture into an execution plan
argument-hint: <feature-or-scope>
---

Use the **master-orchestrator** to run the **Planning phase** for: **$ARGUMENTS**.

Follow this workflow:

1. Confirm that discovery and architecture already exist.
2. Coordinate:
   - scrum-master
   - product-owner-orchestrator
   - tech-lead
3. Produce an execution plan with:
   - phases
   - task breakdown
   - task dependencies
   - blockers
   - handoff points between agents
   - validation checkpoints
   - release prerequisites
4. Keep tasks small, actionable, and ordered.
5. Do **not** implement yet.

Output format:

### Current Phase
Planning

### Agents Involved
List which agents were consulted and why.

### Execution Plan
Break work into ordered tasks.

### Dependencies
List what must happen first.

### Risks / Blockers
List blockers or uncertainties.

### Next Step
Recommend the correct next command.
