# Skill — Design System

## Objective
Garantir consistência visual e velocidade de evolução da interface.

## When To Use
- Criação/ajuste de componentes e padrões de UI.

## Required Inputs
- Biblioteca de componentes/tokens existente.
- Contexto de uso da interface (device, acessibilidade, brand).

## Process
1. Reutilizar padrões existentes antes de criar novos.
2. Aplicar tokens de cor, tipografia, espaçamento e estados.
3. Validar acessibilidade e responsividade.
4. Documentar variantes novas quando necessárias.

## Checklist
- Componente reutiliza padrões/tokens do sistema?
- Estados `default/hover/focus/disabled/error` estão cobertos?
- Contraste e navegação por teclado atendem mínimo de acessibilidade?
- Layout responde bem em mobile e desktop?
- Existe variação desnecessária de estilo?

## Rules
- Evitar one-offs sem justificativa.
- Preferir padrões consistentes a customização local.
