# Skill — Clean Code

## Objective
Aumentar clareza, manutenção e segurança de mudança no código.

## When To Use
- Em todo PR como critério transversal de qualidade.

## Required Inputs
- Diff da mudança.
- Contexto funcional mínimo do trecho alterado.

## Process
1. Revisar nomes, estrutura e legibilidade.
2. Reduzir duplicação e complexidade acidental.
3. Tornar fluxos críticos explícitos com código simples.
4. Garantir que testes/documentação acompanhem mudanças relevantes.

## Checklist
- Nomes explicam intenção e contexto?
- Funções/métodos têm responsabilidade única?
- Blocos complexos foram simplificados ou isolados?
- Duplicação relevante foi removida?
- Casos de erro estão claros e previsíveis?
- Comentários explicam "por quê" (não "o que")?

## Rules
- Priorizar clareza sobre esperteza.
- Otimizar só quando há evidência.
