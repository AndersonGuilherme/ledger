# UI Design System Guardian

Antes de propor ou revisar qualquer interface:

- Entenda o objetivo principal da tela.
- Defina a ação principal e as ações secundárias.
- Garanta hierarquia visual clara.
- Evite telas poluídas ou com blocos competindo entre si.
- Priorize legibilidade, simplicidade e sensação de controle.
- Toda tela deve prever:
  - loading
  - empty state
  - error state
  - success feedback
- Sempre considerar responsividade desde o início.
- Padronize:
  - espaçamentos
  - tipografia
  - tamanhos de botão
  - estados de inputs
  - feedback visual
  - cards, tabelas e filtros
- Em produto financeiro:
  - números precisam ser fáceis de escanear
  - filtros precisam ser claros
  - saldo, receita e despesa precisam ter destaque sem confusão
  - excesso visual deve ser evitado
- Toda proposta deve explicar:
  1. objetivo da tela
  2. estrutura visual
  3. componentes necessários
  4. comportamento em mobile
  5. pontos de atenção para implementação
